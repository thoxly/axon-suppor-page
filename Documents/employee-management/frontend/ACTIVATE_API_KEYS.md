# Активация API ключей

## Текущее состояние

Компонент автоподбора адреса настроен для работы с моковыми данными для тестирования. Для полноценной работы необходимо активировать API ключи.

## Шаги для активации

### 1. Получение API ключей

1. Зайдите на [Яндекс.Облако](https://cloud.yandex.ru/)
2. Создайте платежный аккаунт (если нет)
3. Создайте сервисный аккаунт
4. Получите API ключи для:
   - **Геосаджест API** - для автоподбора адресов
   - **JavaScript API и HTTP Геокодер** - для геокодирования и карт

### 2. Настройка переменных окружения

Добавьте полученные ключи в файл `frontend/.env`:

```env
REACT_APP_GEOSUGGEST=ваш_ключ_геосаджеста
REACT_APP_YANDEX_MAP_JS_API=ваш_ключ_яндекс_карт
```

### 3. Активация проверки API ключей

После получения ключей раскомментируйте проверки в следующих файлах:

#### В `frontend/src/components/common/AddressAutocomplete.js`:

```javascript
// Раскомментируйте этот блок (строки 30-40):
useEffect(() => {
  console.log("AddressAutocomplete: Checking API keys...");
  const keys = checkApiKeys();
  console.log("AddressAutocomplete: API keys status:", keys);

  if (!keys.geosuggest) {
    setApiError("API ключ для Геосаджеста не настроен");
  } else {
    setApiError("");
  }
}, []);
```

#### В функции `fetchSuggestions`:

```javascript
// Раскомментируйте проверку API ключей (строки 60-65):
if (!apiKeys.geosuggest) {
  setSuggestions([]);
  return;
}

// Замените моковые данные на реальный API вызов (строки 75-95):
const results = await getAddressSuggestions(query, { limit: 10 });
setSuggestions(results);
```

#### В `frontend/src/components/common/MapPointPicker.js`:

```javascript
// Раскомментируйте проверку API ключей (строки 50-65):
if (!apiKeys.geocoder) {
  return (
    <Dialog open={open} onClose={onClose} maxWidth="sm" fullWidth>
      <DialogTitle>{title}</DialogTitle>
      <DialogContent>
        <Alert severity="error">
          API ключ для Яндекс Карт не настроен. Невозможно открыть карту.
        </Alert>
      </DialogContent>
      <DialogActions>
        <Button onClick={onClose}>Закрыть</Button>
      </DialogActions>
    </Dialog>
  );
}
```

### 4. Перезапуск сервера

После изменения файла `.env` перезапустите сервер разработки:

```bash
# Остановите сервер (Ctrl+C)
npm start
```

### 5. Проверка работы

1. Откройте консоль браузера (F12)
2. Проверьте, что нет ошибок о недоступности API ключей
3. Протестируйте автоподбор адреса
4. Протестируйте выбор точки на карте

## Функциональность после активации

После активации API ключей компонент будет поддерживать:

- ✅ Реальный автоподбор адресов через Яндекс Геосаджест
- ✅ Геокодирование адресов в координаты
- ✅ Обратное геокодирование координат в адреса
- ✅ Интерактивную карту для выбора точек
- ✅ Получение текущей геолокации пользователя

## Стоимость API

- **Геосаджест**: ~1 рубль за 1000 запросов
- **HTTP Геокодер**: ~1 рубль за 1000 запросов
- **JavaScript API**: бесплатно (с ограничениями)

## Поддержка

При возникновении проблем:

1. Проверьте консоль браузера на наличие ошибок
2. Убедитесь, что API ключи корректны
3. Проверьте лимиты и баланс в Яндекс.Облаке
4. Обратитесь к документации [Яндекс.Карт](https://yandex.ru/dev/maps/)
